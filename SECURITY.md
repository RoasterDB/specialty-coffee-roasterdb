# Security Policy

## Reporting a vulnerability or data concern

We take the integrity and privacy of the RoasterDB data seriously. If you
discover any of the following, please report it privately:

- A security vulnerability in the site ([roasterdb.net](https://roasterdb.net))
  or in anything published in this repository (e.g. the starter notebook).
- A data-privacy concern — a record that exposes information it shouldn't, or
  content that should be removed.
- Any suspected leak of the full (paid) dataset or pipeline source.

**Please email: RoasterDB@proton.me**

Do **not** open a public GitHub issue for security-sensitive reports.

## What to include

- A clear description of the issue and its impact.
- Steps to reproduce (for site/notebook issues) or the affected `product_id` /
  `source_url` (for data concerns).
- Your contact details if you'd like a response.

## Our commitment

- We aim to acknowledge reports within **5 business days**.
- We will keep you informed as we investigate and resolve the issue.
- We will not pursue legal action against good-faith security research that
  respects user privacy and avoids service disruption or data destruction.

## Scope

This repository contains only a **public sample** and documentation — it holds
no credentials, pipeline source, or full dataset. Reports about those (e.g. a
suspected leak) are still very welcome via the email above.

## Roasters

If you're a roaster and want a record corrected or removed, email
**RoasterDB@proton.me** — see also [`CONTRIBUTING.md`](CONTRIBUTING.md).
